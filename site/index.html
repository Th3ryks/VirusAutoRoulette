Built for people who do not want to babysit timers.

The project keeps the useful parts visible and automates the repetitive parts in the background.

01

Multi-account sessions

Each Telegram account gets its own session file and status, loaded from simple ACCOUNTn environment variables.

02

Free spin automation

The worker waits for the next free roulette window, starts the spin, and updates the next timer after success.

03

Daily case handling

Free case checks run alongside roulette, so daily rewards are not missed while accounts stay online.

04

Task requirements

Channel joins, partner clicks, mini-app opens, and cleanup steps are handled through the logged-in Telegram client.

05

Prize claiming

Virus and Stars prizes are claimed to balance, including an inventory sweep on startup and after reward actions.

06

Admin notifications

A Telegram admin bot reports account status and successful reward events without opening the console.

The loop is simple, steady, and visible.

Configure once, authenticate accounts, and let the bot keep cycling through ready rewards.

Configure

Add Telegram API credentials, admin bot token, and as many ACCOUNTn blocks as needed.

Authenticate

Session files are created locally after Telegram login and reused on the next runs.

Automate

The worker tracks roulette and case timers, resolves requirements, spins, opens, and claims.

Observe

The separate dashboard shows account online state, balances, and countdowns with no manual refresh.

Privacy stays local.

Virus Roulette is a self-hosted tool. The public website does not collect analytics, cookies, account data, or Telegram credentials.

Local secrets

Telegram API credentials, bot tokens, and account sessions stay in your own `.env` and `sessions` files.

No hosted accounts

The project author does not receive your Telegram login data, balances, sessions, or dashboard state.

Dashboard privacy

The dashboard is intended for private use because it can expose account status, balances, and reward timers.

Security basics

Never publish `.env`, `*.session`, admin IDs, API hashes, or bot tokens in commits, screenshots, or logs.
